# SFTP Domain Client
Консольное приложение на Java 8 для работы с SFTP-сервером и управления файлом с парами "домен-IP" в JSON-подобном формате

## 0. Исходный код
https://github.com/hugo-obrien/infotecs-test

## 1. Инструкция по сборке проекта
### Требования
- Java: JDK 1.8
- MavenL 3.x

### Процесс сборки
Откройте терминал (командную строку) в корневой директории проекта (где лежит `pom.xml`) и выполните команду

```bash
  mvn clean package
```
### Результат сборки
В директории `target/` будут созданы следующие артефакты:
1. infotecs-1.0-SNAPSHOT-jar-with-dependencies.jar - основной исполняемый файл приложения со всеми зависимостями (JSch)
2. infotecs-1.0-SNAPSHOT-tests.jar - отдельный jar-архив, содержащий скомпилированные тестовые классы

## 2. Инструкция по работе с приложением
### Запуск
Выполните следующую команду в терминале
```bash
    java -jar target/infotecs-1.0-SNAPSHOT-jar-with-dependencies.jar
```

### Интерактивный сеанс
1. **Подключение**: приложение запросит данные для подключения к SFTP-серверу
   - Хост (например localhost или ip-адрес)
   - Порт
   - Логин
   - Пароль
2. **Меню**: После успешного подключения отобразится меню действий:
   - 1 - Получение списка: Выводит все пары, отсортированные по алфавиту по имени домена
   - 2 - Поиск IP по домену: Введите домен, получите соответствующий IP или сообщение об отсутствии
   - 3 - Поиск домена по IP: Введите IP, получите соответствующий домен или сообщение об отсутствии
   - 4 - Добавление пары:
     - Проеряет корректность по IP по стандарту IPv4
     - Проверяут уникальноссть, добавление будет отклонено, если такой домен или IP уже существуют в файле
   - 5 - Удаление пары: Введите домен или IP для удаления соответствующей записи
   - 6 - Завершение работы: Корректное закрытие SFTP-соединяния и вызод из приложения

## 3. Инструкция по запуску тестов и обоснование
### Запуск тестов
Для запуска тестов выполните в корне проекта
```bash
  mvn test
```
Отчеты о прохождении тестов будут сгенерированы в директории `target/surefire-reports/`

### Обоснование тестового покрытия
Тесты разработаны с использованием TestNG и встроенного SFTP-сервера Apache MINA SSHD. Это гарантирует полную изоляцию тестов, их кроссплатформенность (работают без необходимости поднимать Docker или реальный сервер).

Реализованы следующие проверки:
1. Позитивные сценарии
   - Успешное подключение к SFTP-серверу с валидными учетными данными
   - Корректное чтение, добавление и сохранение данных в файл
   - Проверка сортировки
2. Негативные сценарии
   - Попытка подключения с неверным логином/паролем
   - Обработка отсутствия файла
3. Валидация IPv4
4. Проверка уникальности
